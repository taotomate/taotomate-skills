### 1. Diagnosis Phase (Skill Resolution & Scope)

### 2. Action Phase (Parallel Blind Review)
- The **orchestrator NEVER does the code review itself**. It must execute the local `audit_runner.js` script which instantiates auditors in parallel, injects standards, and synthesizes.
- Run the Node command pointing to the target file.

### 3. Verification Phase (Verdict Synthesis & Iteration)
- The `audit_runner.js` script will output a Markdown report with the synthesis of both auditors.
- Present the verdict table as generated by the runner to the user.
- If issues are detected (Confirmed CRITICALs or Real WARNINGs), **ASK** the user: "Should I fix the confirmed issues?".
- **Correction & Re-evaluation Phase**: If the user says YES, delegate the fixes in an iteration (apply strict fixes to the files). After fixing, RE-LAUNCH the `audit_runner.js` audit protocol for re-evaluation. Repeat until `VERDICT: APPROVED ✅` or until the user decides to stop (`ESCALATED ⚠️`).

## Guardrails (Critical Rules)
- **Absolute Block**: Do NOT "git push" or "git commit" after fixing issues WITHOUT running the full parallel re-evaluation first and obtaining `VERDICT: APPROVED`.
- **Warning Classification**:
  - `WARNING (real)`: Causes bug, data loss, or security hole in production.
  - `WARNING (theoretical)`: Requires convoluted or corrupted conditions. Only reported as INFO, does NOT block the verdict and is NOT force-fixed.
- **Convergence**: In Round 2, only re-evaluate if confirmed `CRITICALs` remain. Confirmed real warnings are fixed without re-launching the audit, and suggested/theoretical ones are left untouched. This avoids infinite cycles.

## Troubleshooting

- **`audit_runner.js` not found**: Verify the script exists at `skills/double-blind-review/scripts/audit_runner.js`. If missing, the skill cannot run — escalate to human.
- **Node runtime unavailable**: Check `node --version`. Install Node.js if missing.
- **Infinite re-evaluation loop**: If the iteration limit is hit without `VERDICT: APPROVED`, present the partial results to the user and let them decide whether to merge or escalate.
